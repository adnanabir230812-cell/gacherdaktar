# কৃষিসাথী প্রোজেক্ট অ্যাক্টিভিটি লগ (Krishisathi Project Activity Log)

এই ফাইলটিতে প্রোজেক্টের সকল কাজ, পরিবর্তন এবং পরবর্তী পরিকল্পনার বিবরণ সংরক্ষণ করা হবে, যাতে কোনো কাজের রেকর্ড হারিয়ে না যায়।

---

## [২০২৬-০৬-১৫] চ্যাটবট সার্ভার টাইমআউট ও ব্যাকআপ বাগ ফিক্স

### সমস্যা বিবরণ
১. **Xiaomi MiMo API** সচল থাকা সত্ত্বেও কোডে মাত্র ৩ সেকেন্ডের টাইমআউট থাকার কারণে নেটওয়ার্ক ফ্লাকচুয়েশনে এটি ফেইল করছিল।
২. **FreeLLM API**-এর কি-টি ইনভ্যালিড ছিল এবং এর রেসপন্স টাইম ২২ সেকেন্ডের বেশি ছিল।
৩. চ্যাটবটের রুট কোডে Gemini API keys লুপ করার সময় একটি বাগ ছিল, যা প্রথম কি ফেইল করলে বা সময় কম থাকলে (`timeLimit < 2000ms`) লুপ থেকে সম্পূর্ণ বের হয়ে যেত (`break`), ফলে অন্য সচল Gemini keys আর ট্রাই করা হতো না।

### প্রস্তাবিত সমাধান
১. **MiMo API Timeout** বাড়িয়ে **৮ সেকেন্ড (8000ms)** করা হলো (ইউজার রিকোয়েস্ট অনুযায়ী)।
২. মোট রিকোয়েস্ট টাইম বাজেট **৮.৫ সেকেন্ড থেকে বাড়িয়ে ৯.৮ সেকেন্ড** করা হলো, যাতে Vercel-এর ১০ সেকেন্ড লিমিটের মধ্যে সর্বোচ্চ ব্যাকআপ ট্রাই করা যায়।
৩. Gemini API key লুপের ব্রেকিং থ্রেশহোল্ড **২০০০ms থেকে কমিয়ে ১০০০ms** করা হলো। এর ফলে লিমিটেড সময়েও প্রথম Gemini key-টি ট্রাই করার সুযোগ পাবে।
৪. চ্যাটবটের রেসপন্সে বোল্ড মার্কার (`**`) যাতে সঠিকভাবে হ্যান্ডেল হয় তা নিশ্চিত করা হলো।
